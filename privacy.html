4. Datenerfassung auf dieser Website

Cookies

byteworkz setzt keine Cookies — weder eigene noch Drittanbieter-Cookies. Ein zentraler Cookie-Banner ist deshalb nicht nötig und entfällt bewusst.

Lokaler Speicher (localStorage)

byteworkz nutzt den lokalen Speicher Ihres Browsers (localStorage), um Ihre Dokumente und Tabellen automatisch zu speichern — damit Sie nach einem Browser- Neustart genau da weitermachen können, wo Sie aufgehört haben. Konkret werden dort abgelegt:

  • byteworkz.recent — eine Liste der zuletzt geöffneten Dokumente (Titel, ID, App-Typ, Datum) mit maximal 20 Einträgen
  • byteworkz.docs.<id> — die einzelnen Dokumente und Tabellen mit Ihren Inhalten (Text, Zellwerte, Formeln, eingebettete Bilder als DataURL, Charts, Formatierungen)

Dieser Speicher ist rein lokal und wird zu keinem Zeitpunkt an einen Server übertragen. byteworkz hat keinen Backend-Endpoint, der Dokumente entgegennimmt; der Webserver liefert ausschließlich statische Dateien aus.

Wie Sie diese Daten löschen können:

  • In der App: Auf der Startseite („Hub") jeden Recent- Eintrag einzeln über das ✕-Icon entfernen; das löscht auch das zugehörige Dokument.
  • Browser-weit: In Ihren Browser-Einstellungen unter „Website-Daten" / „Cookies und Daten von Websites" / „Speicher" können Sie alle lokal gespeicherten Daten für byteworkz.byteside.net komplett löschen.

Die Nutzung von localStorage erfolgt ausschließlich, um die primäre Funktion der App (Bearbeiten und Speichern von Dokumenten) bereitzustellen — es findet keinerlei Verarbeitung für Analyse-, Tracking- oder Marketing-Zwecke statt.

Hinweis zur Sprachauswahl auf den Rechtsseiten: Die Umschaltung zwischen DE und EN auf /imprint.html und /privacy.html nutzt einen URL-Parameter (?lang=de bzw. ?lang=en) und keine Cookies oder lokalen Speicher.

Dokumente und Tabellen, die Sie erstellen

Während Sie tippen, formatieren oder Formeln eingeben, lebt Ihr Dokument zunächst im RAM Ihres Browsers und wird parallel in den oben beschriebenen lokalen Speicher geschrieben. Es wird zu keinem Zeitpunkt an einen Server übertragen.

Beim Klick auf Save generiert Ihr Browser lokal eine JSON-Datei (bzw. CSV oder HTML beim Export) und lädt sie auf Ihre Festplatte. Diese Datei wird ebenfalls nicht hochgeladen.

Beim Klick auf Open from file bzw. Open liest Ihr Browser lokal eine Datei von Ihrer Festplatte ein — die Datei verbleibt ausschließlich im Browser und wird zu keinem Zeitpunkt an einen Server gesendet.

Wenn Sie in einem byteDoc-Dokument ein Bild einfügen, wird dieses Bild als DataURL in das JSON-Dokument eingebettet und ist Teil der lokal gespeicherten Daten. Auch hier erfolgt kein Server-Upload.

Schriftarten

byteworkz verwendet den nativen System-Schrift-Stack Ihres Browsers (Inter, -apple-system, Segoe UI, Roboto, …). Es findet keine Verbindung zu Drittanbieter-CDNs wie Google Fonts, fonts.gstatic.com oder unpkg.com statt. Beim Aufruf der Website wird Ihre IP-Adresse ausschließlich an den Hosting-Anbieter Hetzner übermittelt — nicht an Dritte.

Server-Log-Dateien

Der Webserver (Caddy) erfasst und speichert automatisch Informationen in sogenannten Server-Log-Dateien, die Ihr Browser automatisch an mich übermittelt. Dies sind:

  • Browsertyp und Browserversion

  • Referrer URL
  • Uhrzeit der Serveranfrage
  • angefragte URL und HTTP-Status-Code
  • IP-Adresse — vor der Speicherung um das letzte Oktett gekürzt (z. B. 91.107.218.0 statt 91.107.218.123); eine Re-Identifizierung einer konkreten Person ist damit praktisch ausgeschlossen.

Eine Zusammenführung dieser Daten mit anderen Datenquellen wird nicht vorgenommen. Die Erfassung dieser Daten erfolgt auf Grundlage von Art. 6 Abs. 1 lit. f DSGVO; ich habe ein berechtigtes Interesse an der technisch fehlerfreien Bereitstellung und dem Schutz dieser Website vor missbräuchlicher Nutzung.

Speicherdauer: Die maskierten Access-Logs werden regelmäßig (i. d. R. nach 14 Tagen) gelöscht, sobald sie für die genannten Zwecke nicht mehr erforderlich sind. Eine Auswertung zu Werbe- oder Tracking-Zwecken findet nicht statt. Aggregierte Statistiken (z. B. Seitenaufrufe pro Tag) werden über GoAccess aus den maskierten Logs erzeugt; sie enthalten keine personenbezogenen Daten.

4. Data collection on this website

Cookies

byteworkz sets no cookies — neither first-party nor third-party. A central cookie banner is therefore not necessary and is deliberately omitted.

Local storage (localStorage)

byteworkz uses your browser's local storage (localStorage) to automatically save your documents and spreadsheets — so that you can pick up where you left off after a browser restart. Specifically, the following keys are stored:

  • byteworkz.recent — a list of recently opened documents (title, ID, app type, date), capped at 20 entries
  • byteworkz.docs.<id> — the individual documents and spreadsheets with your content (text, cell values, formulas, embedded images as DataURL, charts, formatting)

This storage is purely local and is never transmitted to a server. byteworkz has no backend endpoint that receives documents; the web server serves static files only.

How to delete this data:

  • In the app: on the start page ("Hub"), remove individual Recent entries via the ✕ icon; this also deletes the associated document.
  • Browser-wide: in your browser settings under "Site data" / "Cookies and site data" / "Storage" you can completely delete all locally stored data for byteworkz.byteside.net.

Use of localStorage is exclusively for providing the app's primary function (editing and saving documents); there is no processing for analytics, tracking, or marketing purposes.

Note on language selection on the legal pages: Toggling between DE and EN on /imprint.html and /privacy.html uses a URL parameter (?lang=de or ?lang=en) and no cookies or local storage.

Documents and spreadsheets you create

While you type, format, or enter formulas, your document first lives in your browser's RAM and is also written to local storage as described above. It is never transmitted to a server.

When you click Save, your browser locally generates a JSON file (or CSV / HTML on export) and downloads it to your hard drive. This file is also not uploaded.

When you click Open from file or Open, your browser reads a file locally from your hard drive — the file stays in your browser and is never sent to a server.

If you insert an image into a byteDoc document, that image is embedded as a DataURL in the JSON document and forms part of the locally stored data. Here, too, no server upload takes place.

Fonts

byteworkz uses your browser's native system font stack (Inter, -apple-system, Segoe UI, Roboto, …). It makes no connections to third-party CDNs such as Google Fonts, fonts.gstatic.com, or unpkg.com. When you load the website, your IP address is transmitted only to the hosting provider Hetzner — not to any third party.

Server log files

The web server (Caddy) automatically collects and stores information in so-called server log files, which your browser automatically transmits to me. These are:

  • browser type and version
  • operating system used (from the user-agent)
  • referrer URL
  • time of the server request
  • requested URL and HTTP status code
  • IP address — truncated by the last octet before storage (e.g. 91.107.218.0 instead of 91.107.218.123); re-identification of a specific person is thus practically impossible.

This data is not merged with other data sources. The collection is based on Art. 6(1)(f) GDPR; I have a legitimate interest in the technically error-free provision and protection of this website against misuse.

Retention: The masked access logs are regularly deleted (typically after 14 days) once they are no longer needed for the purposes stated. They are not evaluated for advertising or tracking. Aggregated statistics (e.g. page views per day) generated by GoAccess from the masked logs contain no personal data.
